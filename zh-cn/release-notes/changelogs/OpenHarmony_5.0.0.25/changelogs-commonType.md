# 数据通用类型变更说明

## c1.commonType.1 Asset属性类型变更

**访问级别**

公开接口

**变更原因**

解决在使用分布式数据对象迁移资产时数据覆盖的问题。

**变更影响**

该变更为非兼容性变更。Asset的属性增加undefined类型支持。由于类型不匹配，可能导致代码编译失败。

**起始 API Level**

11

**变更发生版本**

从OpenHarmony SDK 5.0.0.25开始。

**变更的接口/组件**

Asset/数据通用类型（commonType）

**适配指导**

将Asset的必填属性设置为undefined来解决数据覆盖问题。
